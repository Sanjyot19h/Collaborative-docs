# Usage Guide 📖

## Full Contributor Workflow

### 1. Create an Issue
- Go to the **Issues** tab on GitHub
- Click **New Issue**
- Select the **Documentation Update** template
- Describe what you want to add or fix

### 2. Get Assigned
- Wait for a maintainer to assign the issue to you
- Or self-assign if you have access

### 3. Create a Branch
```bash
git checkout -b docs/your-topic-name
```

### 4. Write Your Docs
- Edit files in the `docs/` folder
- Follow Markdown formatting
- Keep content clear and concise

### 5. Commit and Push
```bash
git add .
git commit -m "docs: add usage guide - fixes #ISSUE_NUMBER"
git push origin docs/your-topic-name
```

### 6. Open a Pull Request
- Go to GitHub
- Click **Compare & Pull Request**
- Add a description and link the issue
- Submit for review

### 7. Review Process
- Maintainer reads your changes
- May request edits
- Once approved → **Merged!** ✅
